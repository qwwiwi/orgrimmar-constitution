# Конституция Orgrimmar

**Версия: 3.3.0** (2026-03-01)

Верхнеуровневые правила для всех агентов Family Office AI.

## Что это

Read-only свод правил, который агенты обязаны соблюдать. Конституция стоит выше любых локальных файлов агентов (CONVENTIONS.md, AGENTS.md, SOUL.md). Выше конституции — только прямой приказ принца. Единственное исключение: P1 (безопасность) непереопределяем даже приказом принца (см. PRINCIPLES.md).

## Структура

| Файл | Назначение | Частота изменений |
|------|-----------|-------------------|
| `PRINCIPLES.md` | Ядро: миссия, 4 приоритета, фундаментальные принципы | Редко |
| `CHARTER.md` | Устав: роли, границы, безопасность, эскалация, глоссарий | По мере необходимости |
| `OPERATIONS.md` | Операционная модель: маршрутизация, память, кроны, инварианты | По мере необходимости |
| `CONVENTIONS.md` | Конвенции кода и операций: shell security, git, artifact delivery | По мере необходимости |
| `MISSION.md` | Миссия и стратегические цели | Редко |
| `CHANGELOG.md` | История всех изменений | При каждом изменении |
| `skills/memory-audit/SKILL.md` | Скилл самоаудита памяти агентов | По мере необходимости |
| `proposals/TEMPLATE.md` | Шаблон для предложений от агентов | Не меняется |
| `kaelthas/` | Индивидуальные файлы агента Кельтас | По мере необходимости |

## Иерархия документов (сверху вниз)

```
0. P1 Безопасность (непереопределяем)
1. Прямой приказ принца
2. PRINCIPLES.md (ядро конституции)
3. CHARTER.md (устав)
4. OPERATIONS.md (операционная модель)
5. CONVENTIONS.md (правила кода и процессов)
6. AGENTS.md (индивидуальное поведение)
7. SOUL.md (характер)
```

## Доступ

- **Принц** — единственный кто мержит изменения в `main` (branch protection)
- **Агенты** — читают при старте сессии, предлагают изменения через PR

## Синхронизация

```
GitHub (jasonqween/orgrimmar-constitution)
    ↓ GitHub Action (sa-github-action)
Firebase RTDB (/constitution/)
    ↓ orgbus get constitution/{doc}
Агенты на всех серверах
```

- **GitHub** -- источник правды, принц мержит PR в main
- **Firebase** -- автоматическое зеркало через GitHub Action (sa-github-action)
- **Агенты** -- читают конституцию из Firebase при старте сессии или on-demand
- Локальный git clone (`~/Projects/orgrimmar-constitution/`) -- только для подготовки PR
- Cron `constitution-sync.sh` на серверах -- **DEPRECATED**, заменён Firebase

## Как предложить изменение

1. Создай ветку от `main`
2. Внеси изменения (можно использовать шаблон из `proposals/TEMPLATE.md`)
3. Открой Pull Request в `jasonqween/orgrimmar-constitution`
4. Принц ревьюит и мержит (или отклоняет)
